# 💬 Prompt 调整指南

**让机器人有不同的性格和风格**

---

## 🤔 什么是 Prompt？

Prompt 就是你"对 AI 说的话"。当你对机器人说：

```
你好，请介绍一下你自己
```

这句话就是 Prompt 的一部分。

但更重要的 Prompt 是**系统级 Prompt**，它定义了 AI 的基本性格和行为规则。这个默认是隐藏的，但你可以修改它。

---

## 📁 Prompt 在哪里？

项目中的 Prompt 相关配置在 `.env` 文件中：

```env
# 可以在 .env 中配置 system prompt
# SYSTEM_PROMPT=你是一个友好的助手...
```

> 💡 目前版本主要通过代码配置 prompt。未来会支持单独的 prompt 配置文件。

---

## 🎭 快速开始：改变机器人性格

### 方式 1：修改代码中的 system prompt

找到 `apps/server/src/index.ts` 中的 `messages` 构造部分：

```typescript
const messages: Array<{ role: 'system' | 'user' | 'assistant'; content: string }> = [];
if (this.config.chat.systemPrompt) {
  messages.push({ role: 'system', content: this.config.chat.systemPrompt });
}
```

要设置默认的 system prompt，在 `.env` 中添加：

```env
SYSTEM_PROMPT=你是一个友好的助手，总是尽可能地帮助用户。
```

## 📝 Prompt 模板推荐

### 模板 1：友好助手（默认）

```
你是一个友善的 AI 助手。你应该：
- 友好、耐心地回答问题
- 回答尽量简洁明了
- 如果不懂就说不懂，不要乱编
- 适当使用 emoji 让对话更生动
```

### 模板 2：技术专家

```
你是一个资深技术专家，专注于用通俗易懂的语言解释技术问题。
要求：
- 回答要有技术准确性
- 多用类比和例子解释复杂概念
- 代码示例要清晰、有注释
- 如果不确定，承认不确定性
```

### 模板 3：群聊助手

```
你是一个群聊 AI 助手，专为 QQ 群场景优化。
规则：
- 回答简洁，不啰嗦（群消息不宜太长）
- 如果问题太复杂，先说重点，再问是否需要详细解释
- 可以适度幽默，但不要过度
- 避免太长的代码块，优先文字说明
```

### 模板 4：懒人助手

```
你是一个极简回答专家。
原则：
- 能一个字说清楚的，不说两个字
- 回答控制在 50 字以内
- 除非必要，不展开解释
```

### 模板 5：学习导师

```
你是一个耐心的学习导师，帮助用户理解和掌握知识。
风格：
- 善于提问引导，而不是直接给答案
- 分解复杂问题为简单步骤
- 适时鼓励用户
```

## 🛠️ 如何修改 Prompt

### 第一步：确定你想要什么风格

先想清楚：
- 机器人应该是什么"性格"？
- 回复应该长还是短？
- 要不要用 emoji？
- 适合什么场景？

### 第二步：写出你的 Prompt

参考上面的模板，写一个符合你需求的 prompt。包含：

1. **角色定义**：机器人是什么身份
2. **行为规则**：应该如何回应
3. **边界限制**：什么不该做

### 第三步：测试

启动服务，在群里测试：

```
@机器人 你好
@机器人 什么是量子计算？
@机器人 给我写一个快排算法
```

观察回复是否符合预期。

### 第四步：迭代调整

根据测试结果修改 prompt，直到满意为止。

---

## ⚠️ 修改 Prompt 的注意事项

### 1. 不要太长

Prompt 太长会导致：
- 响应变慢（模型需要处理更多内容）
- 上下文窗口被占用（能记住的对话就少了）

**建议**：Prompt 控制在 500 字以内。

### 2. 明确但不绝对

不要说"你**永远**不能做 X"，改成"你**通常**不做 X"。

这样给模型留点灵活性，它遇到特殊情况时还能变通。

### 3. 测试边界情况

写完 prompt 后，测试一些"不常规"的问题，确保机器人行为符合预期。

### 4. 群聊 Prompt 特别建议

群聊和私聊不同，Prompt 应该考虑：

- **回复要短**：群消息多，太长会刷屏
- **要能打断**：用户可能随时问新问题
- **上下文有限**：群聊历史可能被淹没，不要假设模型能记住太远的内容

## 🔧 进阶：多 Prompt 方案

### 按用途切换

如果你想给机器人多种模式，可以这样设计：

**Prompt A - 闲聊模式**：
```
你是一个友好的闲聊伙伴...
```

**Prompt B - 技术模式**：
```
你是一个技术专家...
```

然后通过命令切换：

```
/ai mode tech    # 切换到技术模式
/ai mode chat    # 切换到闲聊模式
```

> 💡 这需要代码层面的支持，属于进阶功能。

## ❓ 常见问题

### Q: Prompt 改了没效果？

1. 重启服务：`pnpm dev`（Prompt 在启动时加载）
2. 确认 `.env` 格式正确
3. 检查日志是否有加载 prompt 的信息

### Q: 回复太长了？

在 prompt 中加一条：
```
回答尽量简洁，控制在 100 字以内。
```

### Q: 机器人"失忆"了？

确认 `SESSION_MAX_MESSAGES` 不要太小：

```env
SESSION_MAX_MESSAGES=50
```

值太小会导致上下文窗口太小，模型"忘记"之前的对话。
