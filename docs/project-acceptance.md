# 📋 项目执行说明书

## 项目目标

构建一个**QQ 群聊可唤起的 AI 机器人系统**，默认使用阿里云/通义 API 作为主线，面向普通用户开源可部署。

## 当前主线

| 项目 | 说明 |
|------|------|
| **IM 平台** | QQ (通过 NapCatQQ) |
| **AI Provider** | 阿里云/通义 API (默认) |
| **本地模型** | Ollama (可选) |
| **存储** | SQLite (默认) |
| **部署** | Docker Compose |

## 用户体验目标

- 普通用户能在 5 分钟内完成基本配置
- 不需要 GPU 也能运行（API 模式）
- 有 Setup UI 查看状态
- 有 Doctor 命令检查环境
- 文档清晰，遇到问题能自查

## 为什么要有 Setup UI 和 Doctor

| 组件 | 作用 | 用户价值 |
|------|------|----------|
| Setup UI | 状态可视化 | 普通用户不需要看日志也能知道状态 |
| Doctor | 环境检查 | 部署失败时能定位问题 |
| 文档 | 操作引导 | 一步一步引导用户成功 |

## 当前阶段实现重点

2. ✅ 阿里云/通义 API Provider

# ⚠️ 风险与边界说明

## QQ 接入方案风险

| 风险 | 说明 | 缓解 |
|------|------|------|
| NapCatQQ 稳定性 | 非官方方案，可能受 QQ 版本影响 | 关注 NapCatQQ releases 更新 |
| WebSocket 断连 | 网络波动可能导致断连 | 服务端有重连机制 |
| QQ 账号风险 | 使用真人QQ可能有风控风险 | 建议使用专门的小号 |

## API Key 与费用边界

| 项目 | 说明 |
|------|------|
| 阿里云 DashScope | 按 token 计费 |
| 免费额度 | 新用户有免费额度 |
| 成本控制 | 用户需自行管理 API 额度 |

## 本地模型资源要求

| 配置 | CPU | GPU | 内存 |
|------|-----|-----|------|
| qwen2.5:3b | ✅ 可用 | - | 4GB+ |
| qwen2.5:7b | ⚠️ 慢 | 推荐 | 8GB+ |
| qwen2.5:14b | ❌ 慢 | 推荐 | 16GB+ |

## 当前能力边界

| 边界 | 说明 |
|------|------|
| 单实例 | SQLite 不支持高并发 |
| 单 Provider | 同时只用一个 AI Provider |
| 单平台 | 当前只支持 QQ |

## "默认推荐 API"不等于最便宜

| 模式 | 适用场景 |
|------|----------|
| API 模式 | 适合大多数用户，上手快，无需硬件 |
| 本地模式 | 适合有 GPU、想省 API 费用的用户 |

## 最小成功路径

```
1. 注册阿里云账号 → 获取 API Key
2. 下载运行 NapCatQQ → 扫码登录
3. 配置 .env → 填入 API Key 和 QQ_ENABLED=true
4. pnpm dev → 启动服务
5. QQ 群发送 /ai 你好 → 验证成功
```
